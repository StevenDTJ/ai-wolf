# 2026-03-05 狼人杀 8 人局（方案B）+ 女巫规则设计

## 目标
- 扩展狼人杀到 8 人局：2 狼 + 3 民 + 预言家 + 女巫 + 猎人（无守卫）。
- 实现女巫完整夜晚规则与 UI 交互。
- 保持现有流程风格与数据存储方式。

## 核心规则
- 角色配比：**2 狼 + 3 民 + 预言家 + 女巫 + 猎人**。
- 夜晚顺序：**狼人刀人 → 女巫决策 → 预言家验人**。
- 女巫规则：
  1) **救人必须消耗解药**，不用解药不能救。
  2) **毒人必须消耗毒药**，不用毒药不能击杀。
  3) **同一夜三选一**：用解药 / 用毒药 / 都不用（不能同时救毒）。
  4) **女巫决策前必须知道当夜被刀目标**。
- 胜负：狼人全灭则好人胜；好人（含神）人数 ≤ 狼人数则狼人胜（沿用现有逻辑）。

## 状态与数据模型
- 新增角色类型 `witch`，角色映射加入女巫。
- 新增女巫夜晚状态字段：
  - `witchSaveUsed: boolean`
  - `witchPoisonUsed: boolean`
  - `witchDecision: 'save' | 'poison' | 'none'`
  - `witchTargetId: string | null`
- 夜晚结算：
  - `save`：取消狼人击杀（消耗解药）
  - `poison`：额外击杀 `witchTargetId`（消耗毒药）
  - `none`：仅执行狼人击杀

## 流程与组件改动
- 引擎层：
  - 角色池调整为 8 人配置。
  - 增加女巫夜晚处理函数（消费药、生成消息、更新存活）。
  - 结算逻辑按三选一规则。
- UI 层：
  - 女巫行动界面显示“今晚被刀的是 X”。
  - 三选一操作：救 / 毒 / 不用（药用完则禁用）。
  - 女巫卡片展示药品是否已用。

## 错误处理
- UI 禁止非法选择；引擎兜底非法输入按 `none` 处理并记录提示。

## 测试/验证
- 角色分配正确（含猎人、女巫）。
- 夜晚结算：`save` 取消击杀、`poison` 额外击杀、`none` 仅狼人击杀。
- 三选一约束生效。
