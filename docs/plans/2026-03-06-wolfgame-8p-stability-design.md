# 狼人杀 8 人标准局稳定性设计（规则正确性优先）

**日期**: 2026-03-06  
**范围**: 仅 8 人标准局（2 狼、3 村民、1 预言家、1 女巫、1 猎人）  
**目标**: 以“测试 + 实战”双门槛达到可发布稳定性

---

## 1. 目标与非目标

### 目标
- 锁定核心规则，不允许回归破坏。
- 建立可重复验证链路：单元/集成测试 + 剧本回放 + 自动仿真。
- 建立结构化复盘日志，异常可快速定位。

### 非目标
- 不扩展守卫、警长、白狼王等板子。
- 不新增联网房间、匹配系统、排行榜。
- 不优化 UI 风格，仅做稳定性相关必要交互修正。

---

## 2. 发布门槛（双重标准 C）

必须同时满足：
1. **测试门槛**：规则测试 + 黄金回放测试全绿。
2. **实战门槛**：自动对局 20 局连续零异常。

任何一项失败即禁止发布。

---

## 3. 架构策略（风险分层）

### 层 1：规则不变量（发布阻断）
- 夜间结算顺序固定：狼人目标 -> 女巫救/毒 -> 猎人触发（若死亡） -> 天亮播报。
- 角色私有信息隔离：仅角色可见字段可读。
- 胜负判定稳定：任何状态迁移后判定一致。
- 迁移合法：不允许越相位、重复结算、重复死亡。

### 层 2：黄金剧本回放（发布阻断）
- 覆盖关键分支：
  - 女巫救人 / 女巫毒人 / 不用药
  - 预言家存活与死亡分支
  - 猎人夜死 / 日死 / 无可击杀目标
  - 平票、无人出局、连续夜晚推进
- 每个剧本绑定预期事件序列（快照）。

### 层 3：20 局自动仿真（发布阻断）
- 每局检查：
  - 无非法状态（卡死、循环、跳转异常）
  - 无非法角色行为（死人发言/投票/被重复击杀）
  - 可结束并产出明确胜负

### 层 4：结构化复盘日志（发布必备）
- 事件模型：`round`, `phase`, `actor`, `action`, `targets`, `resolution`, `invariantChecks`。
- 失败时输出最小复现路径（最近 N 个事件）。

---

## 4. 关键不变量清单（首批）

1. `alivePlayers` 不含 `isAlive=false` 玩家。
2. 任何玩家死亡后，本局不再进入发言/投票/夜间决策集合。
3. 狼人夜刀目标必须是“当前存活且非狼人”。
4. 女巫解药只能在存在有效被刀目标时生效，且仅一次。
5. 女巫毒药仅一次，目标必须存活且非空。
6. 预言家查验历史是累计历史，不被白天/夜晚误截断。
7. 狼人密聊“昨夜信息”来源于已结算历史，不读取本夜未结算数据。
8. 天亮播报死亡名单与结算结果一致。
9. 投票出局后若触发猎人，猎人结算与播报一致。
10. 胜负判定只基于当前存活阵营，且与规则一致。

---

## 5. 里程碑

### M1（1 周）：规则基线
- 建立不变量测试与核心状态迁移测试。
- 输出：规则测试套件 + CI 阻断规则。

### M2（1 周）：黄金回放
- 新增 10~15 条关键剧本回放。
- 输出：快照基线 + 回放验证工具。

### M3（0.5~1 周）：仿真与发布
- 自动 20 局仿真 + 失败归因。
- 输出：发布检查脚本与门槛文档。

---

## 6. 风险与缓解

- **风险**：AI 输出不稳定导致偶发分支漂移。  
  **缓解**：测试层使用可控输入，仿真层允许随机但强约束不变量。

- **风险**：历史字段来源混乱（未结算 vs 已结算）。  
  **缓解**：统一历史来源优先级：播报事件 > 私有历史 > 本轮草稿态。

- **风险**：规则修复引起 UI 流程连锁。  
  **缓解**：分离引擎层测试与 UI 层测试，先保引擎正确。

- 规则测试：PASS/FAIL

- 阻断项清单：无/有（附 issue）
